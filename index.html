The Execution Pipeline

graph TD A[DataLoader
CSV, Excel, SPSS, Parquet] --> B[AssumptionChecker
Validates distribution constraints] B --> C[MethodSelector
Selects appropriate statistical test] C --> D[ModelFitter
Invokes PyMC or Frequentist Models] D --> E[ResultFormatter
Structures tables and effect sizes] E --> F[ReportBuilder
Renders Jinja2 PDF/DOCX templates]

StatForge implements a robust six-stage asynchronous execution pipeline, optimizing computational loads using `asyncio.to_thread` while maintaining a structured caching layer logic (`joblib.Memory`) for efficient iterative testing.
